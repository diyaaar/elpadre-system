SUMMARY:
Implemented a "Beni Hatırla" (Remember Me) checkbox on the login page to give users explicit control over session persistence. The selection is stored in `localStorage`, and if disabled, a new browser session handles automated sign out via the `AuthContext`.

MOTIVATION:
Supabase defaults to storing the auth session in `localStorage`, which keeps the user logged in across browser restarts. To improve security and user clarity, the user requested an explicit way to opt-in or opt-out of continuous access on a single device without having to manually log out every time.

ARCHITECTURAL IMPACT:
The Client UI Layer (`AuthForm.tsx` and `LoginPage.tsx`) was updated to capture and store the preference. The `AuthContext.tsx` was updated to interpret these flags on bootstrap and terminate the session if a non-persistent session is detected.

None

Low

ROLLBACK STRATEGY:
Revert the additions to `AuthForm.tsx` (the checkbox), the preference storage in `LoginPage.tsx`, and the signout logic in `AuthContext.tsx`.
